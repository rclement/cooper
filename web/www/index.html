Cooper web

Past sessions

Runs entirely client-side, in a Web Worker running a wasm build of the same agent core as the CLI. No server-side proxy — your API key and prompts never leave the browser. With the Local (in-browser) provider, even the model runs in the browser via wllama/WebGPU — no network calls after the model download.

Tools

Built-in tools the agent may call during the conversation.

Context files

Fetched once when added — by URL (subject to CORS) or by path from the Workspace — and forwarded to the agent as extra context. Nothing is added automatically.

Agent instructions

Equivalent to an AGENTS.md — persistent guidance injected into every prompt.

System prompt template

Overrides the whole template (Jinja-style). Leave empty to use the default. Available: agent_instructions, context_files (each with path/ content), current_date, current_time, current_working_dir.

A virtual filesystem, private to this browser (backed by OPFS — the Origin Private File System). Explore, edit, and upload files by hand here, or let the agent read/write it directly via the list_files, read_file, write_file, edit_file, and git_clone tools (toggle them in the Tools panel on the Sessions view).
